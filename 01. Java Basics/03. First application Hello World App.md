## 1. Консольные приложения и потоки данных

Автор упомянул три потока: **Standard Input, Output и Error**.

- **Зачем это бэкендеру:** Даже если ты пишешь сложный сервер, он все равно общается с миром через эти потоки. Например, когда ты видишь логи (текст о работе программы) в терминале — это `System.out` (Standard Output). Если программа падает и пишет красным текстом — это `System.err` (Standard Error).
- **Суть:** Консольное приложение — это база. Оно не тратит ресурсы на отрисовку кнопок и окон, работая напрямую с текстовыми данными.

---

## 2. Жизненный цикл кода (Compilation vs Execution)

Это критически важный момент. Автор подчеркнул: **JVM не понимает `.java` файлы**.

1. **Source Code (`.java`):** Это то, что пишешь ты. Текст для человека.
2. **Compilation (`javac`):** Процесс превращения текста в строгие инструкции для машины. Результат — файл `.class`.

- _Профессиональный нюанс:_ Компилятор проверяет синтаксис. Если ты забыл точку с запятой, программа не просто «не сработает», она даже не «родится» (файл `.class` не будет создан).

3. **Bytecode (`.class`):** Промежуточный язык. Он универсален для Windows, Mac и Linux.
4. **Execution (`java`):** Виртуальная машина (JVM) берет этот байт-код и выполняет его.

---

## 3. Разбор кода: Почему всё именно так?

Автор дал тебе «магическое заклинание»: `public static void main(String[] args)`. Давай расшифруем его по словам:

- **`public`**: Это модификатор доступа. Он означает, что JVM (которая находится «снаружи» твоего файла) имеет право запустить этот метод.
- **`static`**: Позволяет запустить метод без создания объекта класса (эту тему ты глубоко изучишь в ООП, пока просто прими как факт: это «кнопка старта»).
- **`void`**: Означает, что метод просто выполняет работу и ничего не возвращает обратно (например, не вычисляет число для передачи другой программе).
- **`main`**: Это имя-соглашение. JVM ищет именно слово `main`, чтобы понять, откуда начинать.
- **`String[] args`**: Это способ передать твоей программе данные извне прямо в момент запуска (аргументы командной строки).

---

## 4. Именование и Регистр (Case Sensitivity)

Java — очень строгий язык.

- **Классы:** Всегда с большой буквы (`HelloWorld`).
- **Методы и переменные:** С маленькой буквы, каждое следующее слово с большой (`printLine`). Это называется **camelCase** (верблюжий регистр).
- **Ошибка в регистре = Ошибка в коде.** Для Java `main` и `Main` — это два абсолютно разных слова.

---

## 5. Компиляция vs Runtime (Ошибки)

Автор показал два типа проблем:

1. **Compilation Error (Ошибка компиляции):** Твой «черновик» (код) написан с ошибками. Компилятор `javac` говорит: «Я не могу это прочитать, исправляй». Файл `.class` не создается.
2. **Runtime Exception (Ошибка выполнения):** Код написан грамматически верно, компиляция прошла, но **в процессе работы** что-то пошло не так. Например, ты перепутал имя главного метода, и JVM, запустив программу, просто не нашла, с чего начать.

## Профессиональное дополнение от меня:

1. **Почему мы пишем в Блокноте?** Автор прав: IDE (специальные программы для кода) делают слишком много магии за тебя. Понимая, как работает `javac` и `java` в консоли, ты сможешь починить проект, когда магия IDE сломается (а она ломается часто).
2. **Про Semicolon (Точка с запятой):** В Java точка с запятой `;` — это как точка в конце предложения. Без неё Java думает, что ты всё еще продолжаешь одну и ту же мысль, и в итоге «запутывается».
3. **Литералы:** Автор упомянул это слово. Просто запомни: `"Hello World"` — это **String Literal** (строковый литерал). Если ты напишешь `10` — это будет **Integer Literal** (целочисленный литерал). Это просто фиксированные значения, вписанные прямо в код.
