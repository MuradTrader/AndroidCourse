## 1. Выбор версии: LTS против Short-term

Автор говорит о Java 13, но упоминает **LTS (Long Term Support)**.

- **Что это значит для бэкендера:** Oracle выпускает новую Java каждые 6 месяцев. Большинство из них — это «промежуточные» версии. **LTS** — это версии, которые компания обязуется поддерживать и выпускать для них обновления безопасности много лет (например, Java 8, 11, 17, 21).
- **Зачем это бизнесу:** Банки и крупные корпорации не могут переписывать код каждые полгода. Они выбирают LTS-версию и сидят на ней 5-10 лет. Именно поэтому автор говорит, что Java 8 всё еще мега-популярна.
- **Твой выбор:** Для обучения автор советует последнюю версию (на тот момент 13), и это правильно — в новых версиях более удобный синтаксис.

## 2. Oracle JDK vs OpenJDK

Тут автор коснулся важного юридического и технического момента:

- **OpenJDK:** Это полностью открытый исходный код Java. Он бесплатный для всех.
- **Oracle JDK:** Это сборка от компании Oracle на базе OpenJDK, но с их собственными «фишками» по производительности и, самое главное, с платной лицензией для бизнеса (коммерческого использования).
- **Суть:** Для обучения разницы в коде **ноль**. Код, написанный на OpenJDK, будет работать на Oracle JDK и наоборот.

## 3. Инструменты внутри папки `/bin`

Когда автор открыл папку `bin`, он показал два главных файла. Запомни их:

1. **`javac` (Java Compiler):** Твой «завод». Он берет понятный человеку текст `.java` и превращает его в байт-код `.class`.
2. **`java`:** Твой «двигатель». Он запускает виртуальную машину (JVM), которая читает байт-код и выполняет программу.

## 4. Магия Переменных Окружения (Path и JAVA_HOME)

Это самая сложная часть для новичков. Давай объясню на пальцах, зачем это нужно.

Когда ты пишешь в консоли `java`, Windows не знает, где искать эту программу. Она начинает искать её в текущей папке. Если её там нет — выдает ошибку.

### **JAVA_HOME**

Это переменная-указатель. Мы говорим системе: «Запомни, Java установлена вот в этой папке».

- **Зачем это нужно:** Многие программы (например, среды разработки IntelliJ IDEA или серверы баз данных) ищут Java именно по этому имени — `JAVA_HOME`. Если его нет, они не запустятся.

### **PATH**

Это список путей, по которым Windows «пробегается» каждый раз, когда ты вводишь любую команду.

- Добавляя `%JAVA_HOME%\bin` в Path, ты говоришь системе: «Если я введу команду `javac` и ты не найдешь её в текущей папке, загляни в папку `bin` внутри моей Java».
- **Важно:** Автор использовал конструкцию `%JAVA_HOME%\bin`. Это профессиональный подход. Если ты завтра скачаешь Java 21, тебе нужно будет поменять путь только в одном месте (в `JAVA_HOME`), и `Path` обновится автоматически.

## 5. Нюанс с Терминалом

Автор сделал важное замечание: **перезагрузить терминал**.
**Почему это важно?** Когда ты открываешь консоль (CMD или PowerShell), она «фотографирует» текущие настройки системы. Если ты изменил переменные окружения, открытое окно об этом не узнает. Нужно закрыть его и открыть заново, чтобы оно «прочитало» новые данные.

---

### Твое "Профессиональное" дополнение:

Если ты на Windows, проверь команду не только `java -version`, но и `javac -version`. Бывает так, что запускать Java система умеет (JRE уже была), а компилировать — нет. Если обе команды выдают версию — ты настроен идеально.
